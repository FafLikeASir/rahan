## 1. Stack

| Layer | Choice | Note |
|---|---|---|
| Framework | **Next.js 15** (App Router) | No Pages Router |
| Language | **TypeScript** (strict mode) | No JS, no `any` |
| Styling | **Tailwind CSS v4** | Use `@theme inline` in globals.css, not tailwind.config.js |
| Components | **shadcn/ui** (base) | Accessibility + behavior skeleton, overridden with the custom design system |
| Icons | **Lucide React** | Outline, 1.5px stroke. Use sparingly |
| Typography | **Plus Jakarta Sans** (Google Fonts, variable) | Mono accent: JetBrains Mono or Geist Mono |
| Content | **MDX** for case studies | Separates content from component |
| Hosting | **Vercel** | Already in place |
| DS Audit | **Lyse** (`@lyse-labs/lyse`) | DS audit + AGENTS.md generation + MCP server |

## 3. Routing (App Router)

**Two levels only.** No deeper nesting.

| Route | Type | File | Content |
|---|---|---|---|
| `/` | Static | `app/page.tsx` | Home single scroll (6 sections + anchors) |
| `/work/mention` | Static | `app/work/[slug]/page.tsx` | Mention case study |
| `/work/ekonsilio` | Static | `app/work/[slug]/page.tsx` | eKonsilio case study |
| `/work/estorie` | Static | `app/work/[slug]/page.tsx` | eStorie case study |
| `/work/making-this` | Static | `app/work/[slug]/page.tsx` | v1.1 post-launch |

**No `/work` index page.** The `#work` showcase on the home plays that role.

**`generateStaticParams`** in `app/work/[slug]/page.tsx` to pre-render
all case studies at build time.

**Home anchors:** `#hero`, `#work`, `#method`, `#system`, `#about`, `#contact`.

## 4. Server vs Client Components

**Default: Server Component.** `"use client"` only when necessary.

| Component | Server / Client | Reason |
|---|---|---|
| `app/layout.tsx` | Server | Metadata, fonts |
| `app/page.tsx` | Server | Static assembly |
| `<StickyNav>` | **Client** | `useState` for scroll state (transparent → opaque) |
| `<Hero>` | Server wrapper + **Client** for mesh/glass | CSS animations, potentially `useEffect` for the sweep |
| `<HeroMesh>` | **Client** | Animations, potentially mouse interaction later |
| `<WorkCard>` | Server | Static |
| `<MethodSection>` | Server | Static |
| `<SystemShowcase>` | **Client** | Interactive components with states (hover, focus, toggle) |
| `<About>` | Server | Static |
| `<Footer>` | Server | Static |
| `<CaseStudyLayout>` | Server | MDX rendered server-side |
| MDX components | Server | Static content |

**Rule:** never put `"use client"` on a component that does not use
`useState`, `useEffect`, `useRef`, or browser event handlers.

## 5. Styling — Conventions

### 5.1 Token architecture

shadcn/ui brings its own CSS variables (`--background`, `--foreground`,
`--primary`, etc.) in `globals.css`. **We keep and override them** with
Maxime's design system values. We do not create a parallel system.

```css
/* app/global.css — :root (Tailwind v4, not @theme inline for raw CSS vars) */

/* Editorial palette (shadcn override) */
--background: oklch(1 0 0);
--foreground: oklch(0.208 0.042 265.755);  /* slate-900 */
--muted:      oklch(0.984 0.003 247.858);  /* slate-50 */
--border:     oklch(0.929 0.013 255.508);  /* slate-200 */
--accent:     oklch(0.488 0.196 258.5);    /* blue-600 — signature accent */

/* Hero — warm palette (blue/emerald → orange/slate switch, June 2026) */
--hero-bg:           #1f1f1f;   /* hero base background */
--hero-warm-orange:  #fb3706;   /* main blob — signature warm color */
--hero-warm-slate:   #809fb4;   /* secondary blob bottom-left */
--hero-warm-shadow:  #782800;   /* warm shadow (not yet used visually) */

/* Hero nav badge tokens */
--hero-badge-gradient: linear-gradient(99deg, rgba(255,255,255,0.05) 0%, rgba(255,255,255,0.20) 100%);
--hero-badge-glow:     0 0 6px 2px rgba(52,211,153,0.55);  /* emerald dot glow "Available" */

/* Hero glass / overlay */
--hero-grid-size:       40px;
--hero-grid:            rgba(255,255,255,.06);
--hero-glass-a-border:  rgba(255,255,255,.08);
--hero-glass-a-bg:      rgba(255,255,255,.03);
```

**Fundamental rule:** hero tokens are used via `var(--hero-*)` in `inline style` (gradients, blobs). Editorial tokens via Tailwind classes (`bg-background`, `text-foreground`, `border-border`, etc.).

### 5.2 Tailwind rules

- **Use shadcn semantic tokens**: `bg-background`, `text-foreground`,
  `text-muted-foreground`, `border-border`. Never `bg-white` or `text-slate-900`
  hardcoded in components.
- **Custom tokens** (hero, gradient) are used via the corresponding Tailwind
  classes: `bg-hero-bg`, `from-gradient-from to-gradient-to`, etc.
- **`gap-*` instead of `space-y-*`**. Always.
- **`size-*` when width = height.** `size-10` not `w-10 h-10`.
- **`cn()` for conditional classes.** Import from `@/lib/utils`.
- **No `dark:` overrides.** Light only at launch.

### 5.3 shadcn/ui components

- Installed via CLI (`npx shadcn@latest add <component>`).
- **Modified in `src/components/ui/`** to apply Maxime's style.
- The final render must NOT look like default shadcn (red flag 2).
- **Components to install at startup** (anticipated): `button`, `badge`,
  `separator`, `card` (maybe for WorkCards). Rest as needed.
- **Do not install unused components.** The portfolio is simple.

## 6. Typography

```css
/* In globals.css or via next/font */
--font-sans: 'Plus Jakarta Sans', system-ui, -apple-system, sans-serif;
--font-mono: 'JetBrains Mono', ui-monospace, monospace;  /* rare accent only */
```

**Loaded via `next/font/google`** in `src/lib/fonts.ts`:
- Plus Jakarta Sans: weights 400, 500, 600, 700, 800
- Mono: weight 400 only (rare use — System section, spec labels)

**`font-display: swap`** required (performance).

**Type hierarchy:**

| Role | Size | Weight | Color token |
|---|---|---|---|
| H1 (hero) | 48-56px | 800 | `#ffffff` (hero) / `foreground` (page) |
| H2 (section title) | 32-36px | 700 | `foreground` |
| H3 (subtitle) | 22-24px | 600 | `foreground` |
| Body | 16px | 400 | `foreground` |
| Body small | 14-15px | 400 | `muted-foreground` |
| Caption / meta | 12-13px | 500 | `text-tertiary` |
| Mono accent | 13-14px | 400 | `muted-foreground` |

```
Maxime Luet         Work · Method · About · Contact
```

- On the home: links are anchors (`#work`, `#method`, `#about`, `#contact`)

## 9. SEO & Metadata

**`<title>` per page:**
- Home: `Maxime Luet — Product Designer who codes`
- Case study: `[Case title] — Maxime Luet`

**`<meta description>`:** 1 sentence derived from content, per page.

**Open Graph:**
- Image per page (home: dedicated image, case studies: opening visual)
- Use `generateMetadata` in each `page.tsx`

**Technical:**
- `sitemap.ts` generated automatically (Next.js App Router)
- `robots.txt` permissive
- Canonical URLs

**Security headers** (`next.config.ts`):
- `X-Frame-Options: DENY` — anti-clickjacking
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`
- CSP not implemented (GSAP inline styles + SVG data URIs make a strict CSP complex — to address if required)

## 10. Performance

- **Images: `next/image`** everywhere. No native `<img>`.
- **Fonts: `next/font/google`** with `display: swap`.
- **Hero CSS animations:** `transform` and `opacity` only (GPU-composited).
  `will-change: transform` on animated hero elements.
- **`prefers-reduced-motion`**: all animations disabled. Verify.
- **No heavy JS at load.** The hero is pure CSS (no canvas/WebGL at launch).
- **Lazy load images** in case studies (default `next/image` behavior).

---

## 11. Accessibility

- **WCAG AA minimum contrast** across the site (verify accent tokens
  on white background — the classic light mode trap).
- **Visible focus** on all interactive elements (shadcn handles this already).
- **Skip-to-content link** hidden (appears on keyboard focus).
- **ARIA landmarks**: `<header>`, `<main>`, `<footer>`, `<nav>`, `<section>`.
- **Alt text** on all images (including product screenshots).
- **`lang="en"`** on the `<html>`.

## 13. Git & workflow

- **Commits in English**, conventional: `feat:`, `fix:`, `style:`, `docs:`.
- **`main` branch** for production (Vercel auto-deploy).
- **`dev` branch** for work in progress. Merge to `main` when a block is ready.

### Lyse — Design System audit

**When to activate:** once the `@theme inline` tokens and the first 3-4
components are in place (not before — Lyse needs material to audit).

**Workflow:**
1. `npx lyse audit` — check the Health Score after each batch of components
2. `npx lyse agents > AGENTS.md` — regenerate the DS contract after each addition/modification
   of a token or component. This file is read by Claude Code at session start.
3. `npx lyse mcp` — optional, for long Claude Code sessions where the agent
   needs to query tokens in real time.

**Current score: 93/100** (tokens 69, a11y 100, components 100). Remaining findings: documented false positives (display data in `SystemSection`, hero diagonal pattern).

**AGENTS.md** lives at the project root, alongside CLAUDE.md. It is generated,
not written by hand. Do not edit manually — re-run `npx lyse agents`.

## 15. What we do NOT do

Closed list — if Claude Code is tempted to do any of these, it is a stop signal.

- **No Pages Router.** App Router only.
- **No `getServerSideProps` / `getStaticProps`.** That's Pages Router.
- **No `_app.tsx` / `_document.tsx`.** Same.
- **No API routes** (no backend, static site).
- **No database.** Content in MDX + data files.
- **No CMS.** Content is versioned in the repo.
- **No i18n.** English only (texts separated to enable FR later).
- **No dark mode.** Light only at launch.
- **No light/dark toggle.**
- **No `/work` index page.** The `#work` showcase is on the home.
- **No blog.** Out of scope.
- **No contact form.** Email link + LinkedIn + CV PDF.
- **No canvas/WebGL** at launch (pure CSS hero).
- **No framer-motion.** Banned — GSAP is allowed.
- **GSAP**: animation phase only (structure → content → animation). Do not implement
  GSAP animations before structure and content are validated.
  If animations are not specified in the request → propose before implementing.
- **No unused shadcn components.** Install case by case.
- **No `bg-white` or `text-black` hardcoded.** Semantic tokens.
