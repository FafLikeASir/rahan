# Skill Orchestration — Routing Guide

> This file is a routing index for Claude Code. It indicates, for each
> type of task, which skills to use and which documents to read FIRST.
>
> Rule: always read the reference document BEFORE acting. Do not guess
> a decision if it is already made in a doc.

---

## Before any action

1. Read **docs/00-workflow-protocol.md** — session protocol (plan mode, scoring, Figma, Lyse).
2. Read **96-project-structure.md** — architecture and technical constraints.
3. If the project has an **AGENTS.md** (generated by Lyse), read it too.

## Document hierarchy

In case of conflict between documents, priority descends in this order:

1. **96-project-structure.md** — architecture and technical constraints (the final word on "how")
2. **93-creative-direction.md** — the 4 axes and rejection list (the final word on aesthetic "what")
3. **94-information-architecture.md** — structure, labels, template (the final word on "where")
4. **95-hero-visual-spec.md** — hero specification (the final word on the hero)
5. **92-creative-brief.md** — visual direction and scope (framing decisions)
6. **00-brief-global.md** — validated content (texts, cards, sections)
7. **91-brand-discovery.md** — territories and analysis (context, rarely consulted during build)

**Absolute rule:** if a decision is written in one of these docs, it is final.
Do not re-debate it. Do not propose an alternative. Apply it.
If it appears to conflict with another, ask Maxime.

## Installed skills — quick index

| Skill | When to use | When NOT to use |
|---|---|---|
| `design-system` | Tokens, elements, components at system level | Individual pages, content |
| `design-standards` | Production quality: contrast, spacing, mobile, pre-ship | Creative direction, positioning |
| `frontend-component-build` | Building a component: anatomy, variants, states, props, a11y | Textual content, tokens alone |
| `shadcn` | Check/install/override a shadcn/ui component | Custom components without shadcn base |
| `content-and-copy` | Write or edit content (case studies, sections) | Layout, components, tokens |
| `brand-identity` | Color, typography, logo — decisions already made, do not use again unless changing | Current build |
| `information-architecture` | Structure, sitemap, nav — decisions already made | Current build |
| `creative-direction` | Tone, aesthetic axes — decisions already made | Current build |
