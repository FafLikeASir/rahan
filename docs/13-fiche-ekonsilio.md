# Fiche — eKonsilio

> Carte n°3 de la vitrine. Étude de cas À ÉCRIRE.
> Fil narratif : « FROM SCRATCH TO PROD » — différent de Mention (governance).
> Structure forte : un arc en TROIS ACTES (voir plus bas).

## Faits

- **Structure** : eKonsilio — startup de marketing conversationnel (chat en ligne,
  réseaux sociaux, campagnes SMS, génération de leads).
- **Rôle** : Founding UI/UX Designer & Développeur front-end — premier designer.
- **Période** : nov. 2019 – août 2022.
- **Périmètre EXACT de Maxime** (important, red flag 7) :
  UI/UX + design system + développement front des composants (React).
- **Collaborateur** : un développeur fullstack/back + infra (binôme jusqu'à
  sept. 2020). Frontière constante : Maxime = design / DS / front ;
  collègue = back / infra.
- **Croissance équipe** : à partir de sept. 2020, l'équipe grandit (jusqu'à
  ~5 dev fullstack + un PM).

## L'arc en trois actes (le fil de l'étude de cas)

**Acte 1 — Le POC (mars → déc. 2019)**
- Objectif : éprouver l'internalisation de services externes tiers.
- Développé à deux (Maxime front, collègue back/infra). Maxime arrive en renfort
  car « il fallait aller vite ».
- Fast shipping : React + Material UI + CSS custom. PAS de maquettes.
- En PROD dès octobre 2019.
- En parallèle, Maxime teste/expérimente Figma (qui sortait Auto Layout le
  5 déc. 2019 — pile au lancement des fondations design system modernes ;
  les variants n'arriveront qu'à Config 2020).

**Acte 2 — La reconstruction (janv. 2020)**
- « Erased everything, keep the best, avoid POC mistakes. » Départ from scratch.
- Premier vrai projet dans Figma. Design system SANS base (vrai from scratch,
  contrairement à Mention/eStorie qui partaient d'une base).
- Stack : React + Tailwind (front), Storybook (doc), Zeplin (validation design).
- Périmètre Maxime : design system + front de l'app, en solo. Collègue : back/infra.

**Acte 3 — Le passage à l'échelle**
- Arrivée d'un fullstack et d'un PM, l'équipe grandit.
- Maxime se recentre : consolidation du design system, maintenance des composants
  React, doc Storybook, design des maquettes de l'app, support front ponctuel
  (responsive, animations).

## Le produit construit (l'outil métier interne)

Outil métier qui a remplacé les services tiers :
- Interface de chat multi-plateforme (web, réseaux sociaux, SMS) pour les conseillers.
- Gestion clients et conseillers (internes et externes).
- Gestion et personnalisation des intégrations de fenêtres de chat sur sites clients.
- Gestion de leads (datatables filtrables, export…).
- Dashboard reporting (utilisation/engagement/impressions, stats conseillers…)
  avec data viz (KPI cards, charts, datatables).

## Autres réalisations (dossier de compétences)

- UX : A/B testing de la fenêtre de chat (engagement), session tracking conseillers
  (click rage, heatmap, clipboard).
- Outils métier : raccourcis clavier pour accélérer la création de leads,
  sentence library personnalisable pour les conseillers.
- Interviews régulières avec les CSM pour améliorer l'usage métier.
- Workflow Designer ↔ PM ↔ Dev : DoD définie avec le CTO, Single Source of Truth
  (Zeplin), doc Storybook, automatisation Figma/Zeplin/Storybook/Jira/Git.
- Collaboration dev : testing, code reviews, support front (responsive, motion).

## Angle / impact

- **From scratch to prod** + hybridation design/code la plus forte des 3 cartes.
- Histoire d'un fondateur design : fait tout au début, puis structure et délègue
  à mesure que l'équipe se monte (maturité).
- Détail narratif fort : design system construit AU MOMENT où Figma acquérait les
  capacités DS modernes (Auto Layout déc. 2019).
- POC réussi = devenu un vrai produit interne adopté qui a remplacé des
  prestataires externes.
