# .claude/agents/ — Project agents

Specialized agents invocable from custom commands or manually.
Each agent has an `AGENT.md` that defines its sequence and limits.

## Available agents

| Agent | Command | Trigger |
|---|---|---|
| `lyse-auditor` | `/lyse` | End of plan or manually |
| `component-documenter` | `/document ComponentName` | After modifying a component |
| `design-reviewer` | `/design-review` | Before visual implementation |

---

## Root-level linked files (do not move)

These files live at the root due to tool constraints, not by choice:

| File | Generated by | Constraint |
|---|---|---|
| `AGENTS.md` | `npx lyse agents` | Lyse writes to root; read by AI tools from root |
| `LYSE.md` | `npx lyse init` | Lyse writes to root |
| `.lyse.yaml` | Manual | Lyse reads from root by default |
| `lyse.components.json` | Lyse | Path configurable in `.lyse.yaml` |
| `DESIGN.md` | Manual | getdesign.md standard, expected at root |
| `CLAUDE.md` | Manual | Claude Code reads from root |

`.github/workflows/`: GitHub Actions constraint (path not configurable).
